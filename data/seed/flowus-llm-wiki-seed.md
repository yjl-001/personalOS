# Seed Content

这些是建议第一天创建的种子页面和记录，用来让知识库从一开始就有骨架。

## Wiki Pages

### LLM Wiki

- 页面类型：概念
- 一句话摘要：一种由 LLM 持续维护的持久化知识库模式，核心是让知识在 Wiki 中累积，而不是每次查询时从原始资料重新拼装。
- 成熟度：初步成型
- 可信度：中
- 领域：知识管理, AI

正文要点：

- Raw Sources 保留来源。
- Wiki Pages 承载综合后的当前理解。
- Schema / Rules 约束 Agent 如何维护系统。
- Ingest 让新资料进入系统。
- Query 让问题和答案反过来丰富系统。
- Lint 让系统定期自检。

### FlowUS 个人知识库 OS

- 页面类型：主题
- 一句话摘要：以 FlowUS 为知识库本体、以 LLM Agent 为维护者、以 MCP 为读写通道的个人知识管理系统。
- 成熟度：初步成型
- 可信度：中
- 领域：知识管理, AI

### Raw Sources

- 页面类型：概念
- 一句话摘要：知识库中的不可变来源层，保存文章、书籍、视频、播客、对话、日记等原始材料。

### Wiki Pages

- 页面类型：概念
- 一句话摘要：知识库中的综合层，保存被 LLM 维护过的概念、主题、人物、方法、案例和自我知识。

### Questions

- 页面类型：概念
- 一句话摘要：驱动知识库生长的问题层，用于保存研究、决策、自我理解、写作和项目问题。

### Outputs

- 页面类型：概念
- 一句话摘要：知识库的成果层，用于保存文章、memo、PPT、清单、方案、课程和其他输出。

### Log

- 页面类型：概念
- 一句话摘要：知识库的时间线，用于记录 ingest、query、update、lint、output 和 review。

### FlowUS MCP

- 页面类型：概念
- 一句话摘要：让 LLM Agent 可以直接读写 FlowUS 页面、数据库和块的工具通道。
