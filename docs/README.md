# Docs Map

This directory is the canonical design layer for the repository.

## Layout

- `architecture/`: system-facing pages such as dashboard copy and relation design
- `setup/`: live MCP mapping, setup verification, and implementation notes
- `specs/`: source-of-truth local specs such as the database schema
- `workflows/`: operational playbooks for ingest, query, lint, review, and WeRead
- `references/`: failure notes, caveats, and supporting material
