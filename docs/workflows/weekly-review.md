# Workflow: Weekly Review

Use this workflow to maintain the weekly rhythm of the knowledge base so the system keeps growing instead of decaying.

## User Prompt

```text
Run a weekly review on my knowledge base through FlowUS MCP.
```

## Agent Steps

1. Read `Log` entries from the last 7 days.
2. Query `Raw Sources`, `Wiki Pages`, `Questions`, and `Outputs` that were created or updated in the last 7 days.
3. Check whether there are:
   - high-value materials not yet processed
   - answered questions that have not been synthesized
   - Wiki Pages still waiting for verification
   - output opportunities worth turning into drafts
4. Produce a weekly report.
5. Write a `Log` entry and create an `Output` if a deliverable is justified.

## Weekly Report Format

```text
## This Week's Knowledge-Base Changes

## New Sources

## New or Updated Wiki Pages

## Important Shifts in Understanding

## Unresolved Questions

## Next Week's Priorities

## Output Opportunities
```
