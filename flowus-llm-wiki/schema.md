# FlowUS LLM Wiki Schema

这是 FlowUS 中需要创建的核心数据库规格。字段名建议保持一致，因为 Agent 会根据这些名字执行 MCP 读写。

## Relation Map

```text
Raw Sources ──关联──> Wiki Pages
Raw Sources ──关联──> Questions
Wiki Pages  ──关联──> Wiki Pages
Wiki Pages  ──关联──> Questions
Questions   ──关联──> Outputs
Outputs     ──关联──> Raw Sources / Wiki Pages / Questions
Log         ──关联──> Raw Sources / Wiki Pages / Questions / Outputs
```

## 01 Raw Sources

用途：保存不可变的原始资料和处理状态。

建议数据库名：`Raw Sources`

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| 标题 | 标题 | 是 | 资料标题 |
| 类型 | 单选 | 是 | 文章 / 书 / 视频 / 播客 / 论文 / 网页 / 对话 / 日记 / 图片 / 其他 |
| 来源链接 | URL | 否 | 原始链接 |
| 作者 | 文本 | 否 | 作者、机构、说话人 |
| 收集日期 | 日期 | 是 | 资料进入系统的日期 |
| 处理状态 | 单选 | 是 | 未处理 / 阅读中 / 已摘要 / 已入库 / 待复查 / 已归档 |
| 主题标签 | 多选 | 否 | AI、知识管理、健康、写作等 |
| 可信度 | 单选 | 否 | 高 / 中 / 低 / 未评估 |
| 重要性 | 数字 | 否 | 1-5，5 表示最重要 |
| 摘要 | 文本 | 否 | LLM 生成的资料摘要 |
| 关键摘录 | 文本 | 否 | 关键原文摘录或时间戳 |
| 关联 Wiki Pages | 关联 | 否 | 关联到 `Wiki Pages` |
| 关联 Questions | 关联 | 否 | 关联到 `Questions` |
| 处理记录 | 文本 | 否 | Agent 如何处理了这条资料 |
| 最后处理时间 | 日期 | 否 | 最近一次 ingest 时间 |

建议视图：

- `未处理`: 处理状态 = 未处理，按收集日期升序。
- `高价值`: 重要性 >= 4。
- `待复查`: 处理状态 = 待复查。
- `按类型`: 按类型分组。
- `按主题`: 按主题标签分组。

## 02 Wiki Pages

用途：保存知识库中的概念页、主题页、人物页、方法页、案例页和自我知识页。

建议数据库名：`Wiki Pages`

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| 页面名称 | 标题 | 是 | 知识页名称 |
| 页面类型 | 单选 | 是 | 概念 / 主题 / 人物 / 方法 / 案例 / 自我知识 / 决策 / 时间线 / 索引 |
| 一句话摘要 | 文本 | 是 | 让人快速判断页面内容 |
| 成熟度 | 单选 | 是 | 草稿 / 初步成型 / 稳定 / 待重写 / 已废弃 |
| 可信度 | 单选 | 是 | 高 / 中 / 低 / 混合 / 待验证 |
| 领域 | 多选 | 否 | 长期关注领域 |
| 主题标签 | 多选 | 否 | 更细颗粒度的标签 |
| 相关资料 | 关联 | 否 | 关联到 `Raw Sources` |
| 相关问题 | 关联 | 否 | 关联到 `Questions` |
| 相关页面 | 关联 | 否 | 自关联到 `Wiki Pages` |
| 最后更新 | 日期 | 是 | 最近一次内容更新 |
| 推荐复查时间 | 日期 | 否 | 下次复查日期 |
| 待验证点 | 文本 | 否 | 缺证据、需检查、需搜索的问题 |
| 更新摘要 | 文本 | 否 | 最近一次更新改了什么 |

建议视图：

- `全部知识页`: 默认表格。
- `概念地图`: 页面类型 = 概念。
- `主题页`: 页面类型 = 主题。
- `自我知识`: 页面类型 = 自我知识。
- `待重写`: 成熟度 = 待重写。
- `待验证`: 可信度 = 待验证 或 待验证点不为空。
- `近期更新`: 按最后更新倒序。
- `稳定知识`: 成熟度 = 稳定。

## 03 Questions

用途：保存高价值问题。知识库围绕问题生长，而不是围绕收藏生长。

建议数据库名：`Questions`

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| 问题 | 标题 | 是 | 问题本身 |
| 状态 | 单选 | 是 | 未开始 / 探索中 / 已回答 / 持续更新 / 暂停 |
| 问题类型 | 单选 | 是 | 研究 / 决策 / 自我理解 / 写作 / 项目 / 复盘 |
| 优先级 | 单选 | 否 | 低 / 中 / 高 |
| 当前答案 | 文本 | 否 | 当前阶段的综合答案 |
| 相关资料 | 关联 | 否 | 关联到 `Raw Sources` |
| 相关知识页 | 关联 | 否 | 关联到 `Wiki Pages` |
| 输出成果 | 关联 | 否 | 关联到 `Outputs` |
| 下一个问题 | 文本 | 否 | 进一步探索的问题 |
| 是否沉淀为 Wiki | 复选框 | 否 | 是否已写回 Wiki Pages |
| 创建日期 | 日期 | 是 | 问题创建日期 |
| 最后更新 | 日期 | 否 | 最近一次回答或更新 |

建议视图：

- `正在探索`: 状态 = 探索中 或 持续更新。
- `高优先级`: 优先级 = 高。
- `待沉淀`: 是否沉淀为 Wiki = false 且 当前答案不为空。
- `已回答`: 状态 = 已回答。

## 04 Outputs

用途：保存从知识库生成的成果。

建议数据库名：`Outputs`

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| 标题 | 标题 | 是 | 输出标题 |
| 类型 | 单选 | 是 | 文章 / Memo / PPT / 清单 / 方案 / 课程 / 图表 / 对话总结 |
| 状态 | 单选 | 是 | 草稿 / 修改中 / 已完成 / 已发布 / 已归档 |
| 来源问题 | 关联 | 否 | 关联到 `Questions` |
| 来源知识页 | 关联 | 否 | 关联到 `Wiki Pages` |
| 来源资料 | 关联 | 否 | 关联到 `Raw Sources` |
| 发布位置 | URL | 否 | 外部发布链接 |
| 创建日期 | 日期 | 是 | 创建日期 |
| 最后更新 | 日期 | 否 | 最近一次更新 |
| 下一步动作 | 文本 | 否 | 继续修改、发布或复盘 |

建议视图：

- `草稿`: 状态 = 草稿 或 修改中。
- `已完成`: 状态 = 已完成 或 已发布。
- `按类型`: 按类型分组。

## 05 Log

用途：记录知识库的演化。它是 Agent 记忆系统的时间线。

建议数据库名：`Log`

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| 标题 | 标题 | 是 | 日志标题 |
| 时间 | 日期 | 是 | 操作时间 |
| 操作类型 | 单选 | 是 | ingest / query / update / lint / output / review / system |
| 涉及资料 | 关联 | 否 | 关联到 `Raw Sources` |
| 涉及知识页 | 关联 | 否 | 关联到 `Wiki Pages` |
| 涉及问题 | 关联 | 否 | 关联到 `Questions` |
| 涉及输出 | 关联 | 否 | 关联到 `Outputs` |
| 主要变化 | 文本 | 是 | 这次操作改变了什么 |
| 后续动作 | 文本 | 否 | 下一步要做什么 |
| 操作者 | 单选 | 否 | 我 / LLM / 我+LLM |

建议视图：

- `最近日志`: 按时间倒序。
- `Ingest`: 操作类型 = ingest。
- `Query`: 操作类型 = query。
- `Lint`: 操作类型 = lint。
- `System`: 操作类型 = system。

## Field Naming Notes

- 字段名尽量保持中文，和用户日常阅读一致。
- Agent 通过字段名写入时，必须先确认字段存在。
- 如果 FlowUS MCP 返回的是字段 ID，应在 `mcp-integration.md` 中维护字段名到字段 ID 的映射。
