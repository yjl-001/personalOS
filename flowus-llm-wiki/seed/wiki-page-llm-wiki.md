# LLM Wiki

## 当前理解

LLM Wiki 是一种个人或团队知识库模式：LLM 不只是回答临时问题，而是持续维护一个持久化、结构化、互相链接的 Wiki。它把知识管理的重点从“查询时检索”转向“摄入时整合”和“使用后沉淀”。

它和普通 RAG 的关键区别在于：RAG 通常在每次提问时从原始资料中检索片段，再临时拼出答案；LLM Wiki 则让 LLM 在新资料进入系统时就完成摘要、归档、交叉引用、冲突记录和页面更新。这样下一次提问时，Agent 读取的是已经演化过的知识结构，而不是从零开始拼装材料。

在这个模式里，用户的主要工作是提供资料、提出问题、评估方向；LLM 的主要工作是做维护。这个分工很重要，因为人类往往愿意阅读和思考，但不愿意长期做 Wiki 维护中最琐碎的部分。

## 核心要点

- Wiki 是 raw sources 和 query 之间的持久化综合层。
- Ingest 的目标不是摘要一篇资料，而是更新整个知识网络。
- Query 的高价值答案应该沉淀回 Wiki。
- Lint 是必要操作，用于防止知识库变乱、变旧、变散。
- Log 让知识库具备时间线，帮助 Agent 理解最近发生了什么。
- Schema / Rules 让 Agent 成为有纪律的维护者，而不是泛泛聊天助手。

## 支持证据

- LLM 能低成本执行摘要、交叉引用、更新日志和一致性检查。
- Wiki 页面能积累跨资料的综合理解，减少每次查询时的重复拼装。
- 通过 Log 和 Index，Agent 可以在中等规模知识库中较快定位上下文。

## 反对证据 / 争议

- 如果 Agent 无法直接可靠地读写知识库，维护成本会回到用户身上。
- 如果没有安全规则，LLM 可能过度改写旧页面或制造错误关联。
- 如果资料来源质量低，Wiki 会积累看似结构化但未经验证的结论。

## 与我的关系

这个模式适合用来搭建个人知识库，尤其适合长期主题研究、自我理解、读书笔记、项目复盘和输出系统。它的价值不在于收藏更多资料，而在于让每次资料处理和每次提问都能复利。
