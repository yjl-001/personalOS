# Workflow: Ingest

用于处理一条 Raw Source，并把它整合进 Wiki Pages。

## 用户指令

```text
请通过 FlowUS MCP 处理 Raw Sources 里最近一条未处理资料，按 LLM 维护规则完成 ingest。
```

也可以指定资料：

```text
请处理 Raw Sources 中标题为《...》的资料。
```

## Agent 步骤

1. 查询 `Raw Sources`，定位目标资料。
2. 读取资料字段和页面正文。
3. 搜索 `Wiki Pages`，找出相关已有页面。
4. 搜索 `Questions`，找出相关问题。
5. 生成资料摘要。
6. 制定更新计划。
7. 如果影响超过 5 个 Wiki Pages，先请求用户确认。
8. 执行更新。
9. 写入 `Log`。
10. 返回简短报告。

## 摘要格式

```text
### 一句话结论

### 核心观点

### 关键证据

### 对已有知识库的影响

### 新增页面

### 更新页面

### 新问题

### 争议 / 不确定性
```

## 更新规则

- 新概念如果会反复出现，应新建 Wiki Page。
- 只出现一次、价值不高的信息留在 Raw Source 摘要中。
- 旧页面如果被新资料改变，应更新 `当前理解` 和 `更新记录`。
- 旧页面如果只是补充证据，应追加到 `支持证据` 或 `主要来源`。
- 与旧结论冲突的信息应进入 `反对证据 / 争议`，不要静默覆盖。

## 返回格式

```text
已完成 ingest：
- 资料：《...》
- 新增 Wiki Pages：
- 更新 Wiki Pages：
- 新增 Questions：
- 写入 Log：

需要你注意：
- 

建议下一步：
- 
```
