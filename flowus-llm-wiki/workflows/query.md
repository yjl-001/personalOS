# Workflow: Query

用于基于 FlowUS 知识库回答问题，并把有价值的答案写回系统。

## 用户指令

```text
基于我的 FlowUS 知识库，回答这个问题：……
如果答案有长期价值，请沉淀回 Wiki Pages，并写入 Log。
```

## Agent 步骤

1. 搜索 `Questions`，判断是否已有同类问题。
2. 搜索 `Wiki Pages`，读取相关页面。
3. 搜索 `Raw Sources`，读取必要来源。
4. 生成答案，明确：
   - 当前结论
   - 支持依据
   - 反对依据
   - 不确定性
   - 可行动建议
5. 判断是否需要写回：
   - 是否创建或更新 Question
   - 是否创建或更新 Wiki Page
   - 是否创建 Output
6. 执行写回。
7. 写入 `Log`。

## 回答格式

```text
## 当前结论

## 依据

## 不确定性

## 建议行动

## 已写回知识库

## 后续问题
```

## 写回标准

值得写回的答案：

- 复用价值高。
- 改变了已有判断。
- 连接了多个资料或页面。
- 形成了方法、框架、决策或输出草稿。

不一定写回的答案：

- 一次性事实查询。
- 与知识库主题无关。
- 用户明确说只要临时回答。
